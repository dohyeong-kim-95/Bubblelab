# _src — 빌드가 필요한 것들의 소스

정적 HTML로 바로 못 만드는 토이는 소스를 여기 두고, 빌드 결과물만
사이트 폴더에 커밋한다. 이 폴더는 배포되지 않는다.

```
_src/avalon/  →  games/avalon/   (Vite 빌드, ./rebuild.sh 실행)
```

## 관례

- `_src/<이름>/rebuild.sh` — 소스에서 사이트 폴더의 산출물을 재생성하는
  스크립트. 수정 후 이것만 실행하고 결과물을 커밋하면 된다.
- `_src/<이름>/MIGRATION.md` — 어디서 왔고 뭐가 특이한지 기록.
- CI에서 빌드하지 않는 이유: 배포 파이프라인을 의존성 제로로 유지하기
  위해서. 빌드는 로컬에서, 배포는 정적 파일만.
