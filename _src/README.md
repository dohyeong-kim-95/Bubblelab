# _src — 별도 빌드가 필요한 소스

Vite 등 번들 단계가 필요한 프로젝트의 원본을 둡니다. 이 폴더 자체는 배포되지
않으며 생성된 정적 파일만 공개 사이트 폴더에 커밋합니다.

| 소스 | 산출물 | 빌드 |
| --- | --- | --- |
| `_src/avalon/` | `games/avalon/` | `_src/avalon/rebuild.sh` |

## 관례

- `_src/<name>/rebuild.sh`: 의존성 설치, 빌드, 공개 산출물 교체를 한 번에 수행
- `_src/<name>/MIGRATION.md`: 원본 위치와 인프라 이관 기록
- 공개 산출물은 직접 수정하지 않고 항상 소스에서 재생성
- CI의 루트 빌드는 의존성을 최소화하기 위해 `_src` 프로젝트를 다시 번들하지 않음

소스 수정 후 프로젝트 테스트를 실행하고 `rebuild.sh` 결과와 소스를 함께
커밋합니다.

## 공개 전 조건

- `npm audit` 결과와 lockfile을 함께 검토하고 빌드 도구 취약점도 방치하지 않습니다.
- 소스의 환경 변수는 공개 호스트나 비밀이 아닌 빌드 설정만 허용합니다.
- 생성 산출물에 source map, `.env`, API 키, 개발 서버 주소가 포함되지 않았는지
  확인합니다.
- 백엔드가 Archived 상태라면 산출물만 갱신해도 공개 상태를 바꾸지 않습니다.
