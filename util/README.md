# util — 일상 도구

카테고리 상태: **Beta**. 도구별 상태와 데이터 처리가 서로 다릅니다.

<https://util.bubblelab.dev>에서 제공하는 작은 생활 도구입니다. 사이트 루트 카드
페이지는 빌드 시 자동 생성됩니다.

| 경로 | 상태 | 도구 | 데이터 처리 |
| --- | --- | --- | --- |
| `calendar/` | Beta | 달력 이동과 공유 | 서버 저장 없음 |
| `chat/` | Beta | 익명 실시간 채팅 | 메시지 서버 미저장 — 접속자에게 중계만 |
| `convert/` | Beta | 이미지 JPG/PNG 변환 | 이미지가 브라우저 안에서만 처리됨 |
| `fortune/` | Beta | 명식과 오늘의 운세 | 요청 순간 계산, 서버 저장 없음 |
| `passport-pic/` | Beta | 여권사진 규격 크롭·출력 | 이미지가 브라우저 안에서만 처리됨 |
| `photo/` | Beta | 카메라 네컷사진 | 이미지가 브라우저 안에서만 처리됨 |
| `planner/` | Beta | 개인 코드 월간 플래너 | 코드별 서버 저장(당월만), 자가 삭제 가능 |

## 익명 채팅

`chat/`은 `/_chat` WebSocket으로 `ChatDO`(단일 로비)에 접속합니다. 로그인 없이
서버가 임의 닉네임을 부여하며, 닉네임은 브라우저 `localStorage`에만 저장됩니다.
메시지는 서버 어디에도 저장하지 않고 접속 중인 사람에게만 중계합니다 — 접속
시점 이전 대화는 볼 수 없습니다. `/_assets/sticker`의 스티커 팩을 카카오톡
이모티콘처럼 골라 보낼 수 있고(한 번 누르면 미리보기, 한 번 더 누르면 전송),
흰 배경은 `photo/`와 같은 누끼 처리로 브라우저에서 제거해 투명하게 표시합니다.
전송은 이미지 URL이 아닌 `{ pack, n }` 참조로만 이루어져 서버가 허용 목록으로
검증합니다.

남용 방지: 접속은 IP당 분당 20회, 메시지·닉변경은 접속당 10초에 12건으로
제한하고 본문 길이(500자)·Origin·프레임 크기를 검사합니다. 닉네임은 로비 안에서
유일하며, 서버가 30초 간격 ping으로 응답 없는 유령 연결을 정리합니다. 최대 동시
접속 인원(기본 10명)과 로비 초기화(전체 연결 끊기)는 admin 페이지에서 조정할 수
있으며, `ENABLE_CHAT` Worker var로 전체를 끌 수 있습니다.

## 운세

`fortune/`은 `/_fortune/chart`를 호출합니다. Worker가 `KASI_SERVICE_KEY`를 이용해
양력·음력 날짜를 변환하며 키는 브라우저에 노출하지 않습니다. 생년월일시는 서버에
저장하지 않고 요청 순간에만 처리합니다. 입력값은 편의를 위해 해당 브라우저의
`localStorage`에 저장될 수 있습니다.

`/_fortune/chart`는 동일 출처 JSON 요청만 받고 본문 크기와 분당 호출 수를 제한합니다.
KASI 키는 Worker secret에만 있으며 응답이나 정적 JavaScript에 포함되지 않습니다.

결과에는 사주 명식, 오늘의 종합 문구, 재물운·직업운·연애운 상태와 설명이
포함됩니다. 공유 카드는 분야별 운세와 `util.bubblelab.dev/fortune` 주소를 넣어
이미지로 생성합니다.

## 네컷사진과 달력

`photo/`는 사용자가 허용한 카메라 영상을 브라우저 Canvas에서 조합합니다. 촬영
이미지나 완성 이미지를 Bubblelab 서버로 업로드하지 않습니다. 꾸미기 화면에서는
`/_assets` 스티커를 불러와 흰 배경을 브라우저 안에서 제거해 붙일 수 있고,
"프레임 내보내기"는 프레임 정의 ZIP을 사용자 기기에 저장할 뿐 서버로 보내지
않습니다. 내보낸 프레임은 검토 후 `_assets/photo-frame/`에 커밋해야 공용 프레임이
됩니다(자동 업로드 없음). 프레임 선택 화면은 `/_assets/catalog.json`의 공용
프레임을 함께 보여줍니다. 보안 헤더는 카메라를
같은 출처에만 허용하고 마이크·위치 권한은 차단합니다. `calendar/`도 서버 계정이나
일정 저장소를 사용하지 않습니다.

## Planner

플래너는 숫자 6자리와 영문 2자로 이루어진 개인 코드를 사용합니다. 같은 코드를
입력한 PC 화면과 모바일 화면이 `PlannerDO`의 같은 데이터를 봅니다. 모바일도
PLAN/REAL 탭에서 일정 블록을 추가·수정·삭제할 수 있으며(빈 칸 두 번 탭 → 새
블록, 블록 탭 → 수정 시트), TODO·블록 편집은 동시 편집 충돌을 줄이기 위해 전체
저장이 아닌 항목 단위 PATCH로 동기화합니다.
브라우저가 직접 Durable Object를 호출하지 않고 `/_planner/*` 세션 API를 사용하며,
운영에는 `PLANNER_SESSION_SECRET` 또는 대체 가능한 관리자 세션 secret이 필요합니다.

공개 운영 정책 (로그인 화면에도 고지):

- **소유권**: 코드를 아는 사람을 소유자로 봅니다. 코드가 곧 열쇠이므로 공유하면
  통제할 수 없고, 로그인 화면에서 무작위 코드 생성을 권장합니다.
- **복구**: 분실한 코드는 복구할 수 없습니다. 새 코드로 다시 시작합니다.
- **보존**: 서버는 KST 현재 달 데이터만 유지하고 지난 달은 자동 삭제합니다.
- **삭제**: 화면의 "Delete all data" 버튼(`DELETE /_planner/data`)으로 소유자가
  서버 데이터를 즉시 삭제할 수 있습니다.
- **추측·남용 방지**: 로그인은 15분당 5회, 데이터 쓰기는 분당 60회로 제한하고
  본문 크기(600KB)와 동일 출처를 검증합니다. 로그인 응답은 데이터 존재 여부를
  드러내지 않습니다.
