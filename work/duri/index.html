Duri는 메신저가 아니라 두 사람을 위한 개인 라이프 시스템입니다. 주고받는 대화와 사진, 일정·장소·메모가 매일의 생활 속에서 자동으로 남고, 하나의 타임라인으로 구조화되어 평생 보존됩니다.

Status

Phase 4 — 구현 준비 완료 (설계 게이트 통과)

  • 설계 확정 — PRD · 데이터 모델 · ADR 001~008
  • 저장소 라이터(Storage Writer) 구현 게이트 통과
  • 인증/세션(Auth·Session) 구현 게이트 통과
  • HTTP 인증 엔드포인트 · 사진 업로드 영속화
  • 백업/내보내기(Export-as-storage) 마무리

라이브 배포(duri.bubblelab.dev)는 두 사람만 접근하도록 초대 코드와 기기 등록으로 통제됩니다. 이 미리보기 페이지는 진행 상황 공유용입니다.

Approach

보존을 우선한 설계

모든 데이터는 사람이 읽을 수 있는 형식으로 DuriStorage/ 파일시스템에 남습니다 — 월별 metadata.json과 원본 미디어. SQLite는 색인·캐시일 뿐, 진실의 원천은 파일입니다. 자동 이벤트 묶음이나 AI 분류는 하지 않고(ADR-002·003), 기록을 있는 그대로 쌓아 저장소 구조 자체가 내보내기 형식이 되도록 했습니다(ADR-007).
