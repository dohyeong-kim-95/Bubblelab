# daonfit — 다온핏 브랜드 홈페이지 (외주 시안)

상태: **작업 중**. work.bubblelab.dev/daonfit 에서 비밀번호 입력 후 확인.

시안 방향이 확정되어 "따뜻한 크림" 안(구 `v1/`)을 루트로 승격했습니다.
비교용으로 두던 다른 시안(브라운·갤러리 화이트)은 삭제했습니다.

3D프린팅 생활 맞춤 소품 브랜드 "다온핏"(스마트스토어 `samyong_2`,
인스타그램 `@daonfit`)의 자체 홈페이지 시안입니다. 구매는 스마트스토어로
연결하며 이 사이트는 서버에 아무것도 저장하지 않습니다.

- `_work_assets/`의 로고·상품 사진 권리는 다온핏에 있습니다. 무단 사용 금지.
  현재 상품 사진은 스토어 스크린샷 기반 임시본으로, 인계 전 원본으로 교체합니다.
- 브랜드 슬로건: "좋은 것들이 모여, 너에게 꼭 맞게"
- 홈은 히어로 → **커스텀 제작 안내("이런 것도 만들어줘!")** → 판매 상품 순서.
  커스텀 제작이 일반 쇼핑몰과 가장 다른 지점이라 상단으로 배치했습니다.
- 문의 Q&A: `qna.html`(고객)과 `admin/`(관리자)이 `/_workqna/daonfit` API를
  사용합니다. work 게이트 세션이 있어야 읽기·쓰기가 가능하며, 문의는 닉네임과
  내용만 받고(개인정보 미기재 안내) `WorkQnaDO`에 프로젝트별로 저장됩니다.
  주의: 이 API는 bubblelab Worker에 있으므로 정적 인계 시에는 별도 백엔드로
  옮기거나 스마트스토어 문의로 대체해야 합니다.
- 구매 후기·상품 문의: 품목별 상세페이지(`goods/*.html`)가 `reviews.js`로
  자기 상품의 **리뷰**와 **문의(Q&A)**를 함께 보여줍니다. 네이버에서 동기화된
  항목엔 **네이버 마크(N)**, 다온핏 자체 등록분엔 `다온핏` 배지를 붙여 출처를
  구분합니다(mock은 두 출처 혼합). **"후기 남기기" 버튼**으로 방문자가 후기를
  직접 남길 수 있고(별점+내용), 작성 후기는 다온핏 배지로 노출되며 동기화 시에도
  보존됩니다. 백엔드는 네이버 커머스 API 연동이며 **현재는 mock 데이터**로
  동작합니다. 자체 완결을 위해 백엔드 원본을 `_backend/reviews.js`에 함께 넣어
  두었습니다(리포의 `_infra/reviews.js`와 독립된 동일 사본). 실서비스 전환
  절차·주의사항은 `REVIEWS.md` 참고.
- 자체 완결: 이 폴더는 `_shared` 등 리포 공용 리소스에 의존하지 않고 폴더 내부
  리소스만으로 구동됩니다. 인계 시 이 폴더를 통째로 전달하고 리포에서 삭제합니다.
