# _shared — 공용 에셋

모든 서브도메인에서 `/_shared/파일명` 으로 접근되는 파일들.
(워커가 이 경로만 사이트 프리픽스 없이 서빙한다)

## realtime-client.js — 범용 실시간 연결

`/_rt/<namespace>` Durable Object에 WebSocket으로 연결한다.

- JSON 경로 `get`, `set`, `update`, `remove`
- 경로별 실시간 구독
- 연결 종료 시 기록할 값 등록
- 연결이 끊기면 구독과 종료 작업을 복구하며 재접속

## multiplayer-room.js — 멀티플레이 방

게임별 규칙과 분리된 공통 방 수명주기 모듈.

- 브라우저별 플레이어 ID와 닉네임 복구
- 혼동하기 쉬운 I/O를 제외한 영문 6자리 코드
- 방 생성·입장·퇴장·강제 퇴장
- 최대 인원과 닉네임 중복 검사
- 온라인 상태와 방장 승계
- 오래된 방과 비공개 데이터 정리

게임은 별도 namespace와 `rooms`, `privateData`, `actions` 하위 구조를 사용한다.

## share.js — 공유 버튼

토이의 `</body>` 직전에 한 줄:

```html
<script defer src="/_shared/share.js"></script>
```

- 우하단에 반투명 "📤 공유" 버튼이 뜬다
- 모바일: OS 공유 시트 (카카오톡·문자 등 포함) / 데스크톱: 링크 복사 + 토스트
- 전체 화면이 클릭 영역인 토이에도 안전 (이벤트 전파 차단)

### 자랑 문구

토이 스크립트에서 전역 훅 하나만 정의하면 공유 내용에 문구가 실린다:

```js
window.blShareText = () => `내 반응속도는 ${best}ms! ⚡ 도전해보세요`;
// 함수 대신 문자열도 가능. 없으면 링크만 공유.
```

클립보드 복사 시에는 `문구\nURL` 형태로 복사된다.

### 나중에: 카카오톡 전용 버튼

공유 시트 안 거치고 바로 카톡을 띄우려면 카카오 개발자 앱 등록 +
JavaScript 키가 필요하다. 키가 생기면 share.js에 SDK 연동을 추가할 것.
