# Bubble Pop Idle 장기 진행 설계 리서치

- 작성일: 2026-07-14
- 대상: `idle.bubblelab.dev/bubble-pop`
- 상태: V2 구현 기준 문서. 2026-07-14에 첫 계층 재조정과 버블 압축이 적용됐다.

## 1. 현재 관찰

실제 플레이 테스트에서는 시작부터 마지막 생산기 해금까지 약 **10~20분**이
걸렸다. 7일 시즌 게임의 첫 계층으로는 너무 짧다. 문제는 단순히 숫자가 빨리
오르는 것이 아니라, 플레이어가 짧은 시간 안에 화면에 보이는 콘텐츠를 모두
확인해 더 이상 기다릴 이유를 잃는다는 점이다.

따라서 다음 두 문제를 별도로 해결해야 한다.

1. 현재 생산기 계층을 너무 답답하지 않은 범위에서 완만하게 늘린다.
2. 모든 생산기를 해금한 순간 게임이 끝나지 않고 새로운 규칙의 계층이 열린다.

## 2. 다른 방치형 게임의 해결 방식

### 2.1 성장 곡선을 시뮬레이션한다

Anthony Pecorella의 방치형 게임 설계 자료는 생산량, 기하급수 가격, 묶음 구매,
구매 최적화와 프레스티지 주기를 별도의 곡선으로 모델링한다. 핵심은 가격만
일괄적으로 올리는 것이 아니라, 플레이어가 다음 의미 있는 결정을 만나는
시간을 목표로 두고 여러 플레이 방식의 도달 시간을 계산하는 것이다.[1][2]

Bubble Pop Idle도 최소한 다음 세 경로를 따로 시뮬레이션해야 한다.

- 적극적 플레이: 버블 터치와 최적 구매를 계속 수행
- 혼합 플레이: 몇 분 조작한 뒤 주기적으로 복귀
- 방치 플레이: 생산기를 산 뒤 오프라인 보상 위주로 진행

### 2.2 첫 계층의 끝에서 프레스티지를 연다

Cookie Clicker는 Ascension으로 일반 진행을 초기화하는 대신 Heavenly Chips와
영구 업그레이드를 남긴다. AdVenture Capitalist도 Angel Investor를 얻고 일반
사업 진행을 다시 시작한다.[3][4] 같은 콘텐츠를 반복하더라도 이전보다 빠르게
통과하거나 새로운 업그레이드를 고르게 만들어 첫 계층의 수명을 늘린다.

다만 단순히 `초기화 → 전체 생산 ×N`만 제공하면 반복 노동이 되기 쉽다.
Bubble Pop Idle은 이미 매주 전체 초기화가 있으므로, 시즌 내부 프레스티지를
도입하더라도 새로운 선택이나 기능을 함께 제공해야 한다.

### 2.3 규칙이 달라지는 다음 계층을 전개한다

Antimatter Dimensions는 한 생산 계층을 완료하면 Infinity, Eternity, Reality와
같은 다음 프레스티지 계층을 열고, 각 계층에 별도 재화와 업그레이드 구조를
둔다.[5] 이전 콘텐츠의 끝이 전체 게임의 끝이 아니라 새로운 규칙의 시작이다.

Trimps 역시 지역 진행, 도전 과제, 영구 특성, 자동화를 순차적으로 열어 후반의
판단을 단순 구매량에서 빌드와 목표 선택으로 바꾼다.[6]

### 2.4 기다림에는 다음 변화가 예고되어야 한다

방치형 게임 참여 연구에서는 낮은 스트레스, 성취감, 지속적인 성장과 복귀 후
확인할 변화가 장르의 중요한 매력으로 관찰됐다.[7] 숫자만 더 비싸게 만들면
성장이 느려진 것이 아니라 콘텐츠가 멈춘 것으로 받아들여질 수 있다.

따라서 긴 정체 구간 앞에는 다음 생산기나 계층, 새 재화, 새 선택 중 적어도
하나를 명확히 예고해야 한다.

## 3. Bubble Pop Idle에 적용할 결론

### 3.1 첫 계층 목표

현재 10~20분인 전체 생산기 해금 시간을 첫 조정에서는 아래 범위로 옮긴다.
정확한 값은 시뮬레이션과 플레이 테스트 후 확정한다.

| 플레이 방식 | 첫 계층 전체 해금 목표 |
|---|---:|
| 적극적 플레이 | 45~90분 |
| 혼합 플레이 | 2~4시간 |
| 주로 방치 | 6~12시간 |

초반 첫 구매 속도와 첫 5분의 재미는 현재 수준을 유지한다. 중후반 생산기의
해금 간격과 비용·생산 곡선부터 벌려 플레이 시작부터 기다리게 만들지 않는다.

### 3.2 두 번째 계층: 버블 압축

모든 기본 생산기를 해금하면 **버블 압축** 계층을 연다. 이것은 최종 명칭이
아닌 작업명이다.

- 해금 조건: 모든 기본 생산기 해금 + 누적 버블 기준 충족
- 새 재화: `압력`
- 획득 방식: 전체 자동 생산량의 제곱근에 비례해 수동 초기화 없이 자동 생성
- 용도: 희귀 버블, 직접 터치, 자동 생산, 오프라인 효율 중 선택 강화
- 제한: 한 시즌에 모든 경로를 완성할 수 없도록 선택 비용 설계
- 목표: 같은 시작점에서도 서로 다른 빌드와 최종 점수가 나오게 한다

첫 계층은 `무엇을 살 수 있는가`를 배우는 구간이고, 두 번째 계층은 `어디에
집중할 것인가`를 결정하는 구간이 된다.

### 3.3 7일 시즌의 권장 흐름

| 시점 | 기대 경험 |
|---|---|
| 0~5분 | 직접 터뜨리기, 첫 자동 생산, 빠른 보상 |
| 5~30분 | 생산기 확장과 25개 배수 달성 |
| 45~90분 | 기본 생산기 전체 해금, 버블 압축 예고 및 진입 |
| 2~12시간 | 압력 생산과 첫 빌드 선택 |
| 2~4일 | 선택 경로 강화, 희귀 버블과 자동화 확장 |
| 5~7일 | 새 콘텐츠 해금보다 시즌 점수 최적화와 순위 경쟁 |

## 4. 피해야 할 해결책

- 모든 가격을 같은 비율로 크게 올리기
- 실제 날짜가 지나야만 다음 기능이 열리는 강제 시간문
- 설명 없는 생산 효율 소프트캡
- 첫 계층과 같은 생산기만 계속 추가하기
- 초기화 보상이 단순 전체 생산 배수뿐인 프레스티지

이 방식들은 플레이 시간을 늘릴 수는 있지만 새로운 판단을 만들지 못한다.

## 5. 구현 전 검증 항목

1. 현재 수식으로 적극적·혼합·방치 플레이의 해금 시간을 재현한다.
2. 첫 5분은 유지하면서 마지막 세 생산기의 도달 시간만 완만하게 벌린다.
3. `NEXT DOUBLE`과 다음 생산기뿐 아니라 버블 압축 해금도 미리 보여준다.
4. 두 번째 계층에서 최소 두 개의 유효한 빌드가 나오는지 계산한다.
5. 개발자 초기화와 가속 시뮬레이션으로 7일 전체 곡선을 반복 검증한다.

## 6. 성공 기준

- 신규 플레이어가 첫 5분 안에 최소 두 번 의미 있는 구매를 한다.
- 적극적으로 플레이해도 기본 계층 전체 해금에 최소 45분이 걸린다.
- 기본 계층을 마치기 전에 다음 계층의 존재와 조건을 알 수 있다.
- 기본 계층을 마친 직후 새로운 재화와 선택이 생긴다.
- 2일 차 이후에도 단순히 같은 생산기의 수량만 올리지 않는다.
- 마지막 날에는 콘텐츠 부족이 아니라 빌드 최적화가 주요 목표가 된다.

## 7. V2 구현 기록

- 밸런스 시뮬레이터: `_infra/idle-balance.mjs`
- 이론적 최단 전체 해금: 약 35.1분. 기존 실제 플레이와 시뮬레이터의
  차이를 적용하면 적극적 플레이 약 60~120분을 예상한다.
- 기본 업그레이드 `안정된 흐름`은 레벨당 자동 생산 ×1.6에서 ×1.45로
  조정했다. 12레벨 누적 배수는 약 281배에서 약 86배로 낮아진다.
- 첫 세 생산기는 유지하고 연구소·구름·반응로·바다의 곡선만 완만하게 벌렸다.
- 모든 생산기를 하나 이상 보유하면 초기화 없이 압력 계층이 열린다.
- 압력 경로: 자동 생산 ×1.35, 직접·희귀 보상 ×1.75, 오프라인 효율 +25%p,
  압력 생산 ×1.6. 각 경로는 반복 구매할 수 있고 비용은 4배씩 증가한다.
- 기존 V1 저장은 버블·생산기·주간 기록을 유지한 채 V2로 마이그레이션한다.
- 오프라인 제한은 기존과 동일하게 최대 24시간이며 압력에도 같은 제한을 쓴다.
